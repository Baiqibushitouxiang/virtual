# 一期接口文档

## 1. 文档说明
- 适用范围：一期改造涉及的场景与资源、模型数据绑定、展板、设备数据、告警规则、OPC UA 接入接口。
- 返回结构：后端大部分业务接口统一返回 `Result` 对象。
- `Result` 结构：

```json
{
  "code": "200",
  "msg": "操作成功",
  "data": {}
}
```

- 关键兼容说明：
  - 场景 JSON 支持新旧两种格式。
  - `scene_assets.texture_info` 现在承载“场景元数据”，包含 `version / textures / bindings`。
  - 展板不再是主数据绑定入口，主绑定在场景编辑器的模型详情面板完成。
  - 告警当前通知实现为服务端控制台打印。

## 2. 场景与资源

### 2.1 上传场景
- 方法：`POST /api/scenes/upload`
- 用途：上传 JSON/GLB/GLTF 场景资源，并可更新已有场景。
- 请求参数：
  - `file`：场景文件，必填
  - `name`：场景名称，可选
  - `description`：场景描述，可选
  - `id`：场景 ID，可选；传入时表示更新已存在场景
- 成功返回：`SceneAsset`
- 关键约束：只支持 `json|glb|gltf`

### 2.2 场景列表
- 方法：`GET /api/scenes`
- 用途：获取场景资源列表
- 返回：`SceneAsset[]`

### 2.3 场景详情
- 方法：`GET /api/scenes/{id}`
- 用途：获取单个场景资源详情
- 返回：`SceneAsset`
- 关键字段：
  - `fileType`
  - `path`
  - `url`
  - `textureInfo`：当前作为场景元数据存储字段

### 2.4 删除场景
- 方法：`DELETE /api/scenes/{id}`
- 用途：删除场景资源及其存储文件

### 2.5 保存场景元数据
- 方法：`POST /api/scenes/{id}/textures`
- 用途：保存场景纹理缓存、模型绑定等元数据
- 请求体：字符串或 JSON 字符串
- 推荐新格式：

```json
{
  "version": "1.0.0",
  "textures": {
    "scene-model-1": "data:image/png;base64,..."
  },
  "bindings": [
    {
      "sceneId": 11,
      "modelId": "scene-model-1",
      "modelName": "主泵模型",
      "deviceId": 1,
      "deviceCode": "DEV-001",
      "deviceName": "主泵设备",
      "dataType": "temperature",
      "ruleStatus": 1
    }
  ]
}
```

## 3. 模型数据绑定

### 3.1 查询模型绑定
- 方法：`GET /api/model-bindings`
- 用途：查询模型与设备数据项的绑定关系
- 查询参数：
  - `sceneId`：可选；传入时按场景查询
- 返回：`ModelDataBinding[]`

### 3.2 新增模型绑定
- 方法：`POST /api/model-bindings`
- 用途：新增模型绑定
- 请求体字段：
  - `sceneId`
  - `modelId`
  - `modelName`
  - `deviceId`
  - `deviceCode` 可由服务端回填
  - `deviceName` 可由服务端回填
  - `dataType`
  - `ruleStatus`
- 关键逻辑：同一 `sceneId + modelId` 应视为一条绑定，重复保存时更新而不是新增脏数据

### 3.3 更新模型绑定
- 方法：`PUT /api/model-bindings/{id}`
- 用途：更新已有绑定

### 3.4 删除单条绑定
- 方法：`DELETE /api/model-bindings/{id}`
- 用途：按主键删除绑定

### 3.5 按场景与模型删除绑定
- 方法：`DELETE /api/model-bindings?sceneId={sceneId}&modelId={modelId}`
- 用途：删除指定场景下指定模型的绑定关系

## 4. 展板管理

### 4.1 查询展板
- 方法：`GET /api/data-panels`
- 用途：获取展板列表
- 关键说明：展板负责展示与布局，主数据绑定不在这里完成

### 4.2 新增展板
- 方法：`POST /api/data-panels`
- 用途：创建展板
- 请求体字段：
  - `name`
  - `description`
  - `status`

### 4.3 更新展板
- 方法：`PUT /api/data-panels/{id}`
- 用途：更新展板名称、描述、状态等

### 4.4 删除展板
- 方法：`DELETE /api/data-panels/{id}`

### 4.5 绑定模型到展板
- 方法：`POST /api/data-panels/{id}/bind-model`
- 用途：将展板绑定到场景中的某个模型
- 请求体：

```json
{
  "modelId": "scene-model-1",
  "modelName": "主泵模型",
  "modelType": "scene"
}
```

### 4.6 解除展板模型绑定
- 方法：`POST /api/data-panels/{id}/unbind-model`

### 4.7 更新展板位置
- 方法：`PUT /api/data-panels/{id}/position`
- 请求体：

```json
{
  "position": "{\"top\":\"80px\",\"left\":\"20px\"}"
}
```

## 5. 设备管理

### 5.1 注册设备
- 方法：`POST /api/devices`
- 用途：创建设备档案
- 请求体：

```json
{
  "name": "主泵设备",
  "description": "泵房主泵"
}
```

### 5.2 查询设备列表
- 方法：`GET /api/devices`
- 查询参数：
  - `userId` 可选
  - `status` 可选

### 5.3 查询设备详情
- 方法：`GET /api/devices/{id}`

### 5.4 按设备编码查询
- 方法：`GET /api/devices/deviceId/{deviceId}`

### 5.5 更新设备
- 方法：`PUT /api/devices/{id}`

### 5.6 删除设备
- 方法：`DELETE /api/devices/{id}`

### 5.7 绑定用户
- 方法：`POST /api/devices/{id}/bind`

### 5.8 解绑用户
- 方法：`POST /api/devices/{id}/unbind`

### 5.9 启停状态
- 方法：`PUT /api/devices/{id}/status?status=1`

### 5.10 证书相关
- `POST /api/devices/{id}/certificate`
- `GET /api/devices/{id}/certificate`
- `POST /api/devices/{id}/certificate/revoke`

## 6. 设备数据

### 6.1 写入设备数据
- 方法：`POST /api/device-data`
- 用途：写入设备实时数据，并触发告警评估
- 请求体：

```json
{
  "deviceId": "DEV-001",
  "dataType": "temperature",
  "value": 82.5,
  "unit": "C"
}
```

- 关键逻辑：
  - 保存数据
  - 更新设备在线状态
  - 推送 WebSocket 数据
  - 执行告警规则判断

### 6.2 查询最新数据
- 方法：`GET /api/device-data/latest/{deviceId}?limit=10`

### 6.3 查询历史数据
- 方法：`GET /api/device-data/history/{deviceId}`
- 查询参数：`dataType`、`startTime`、`endTime`、`limit`

### 6.4 查询统计数据
- 方法：`GET /api/device-data/stats/{deviceId}`
- 查询参数：`dataType`、`startTime`、`endTime`

### 6.5 查询设备数据量
- 方法：`GET /api/device-data/count/{deviceId}`

### 6.6 清理历史数据
- 方法：`DELETE /api/device-data/clean?daysToKeep=30`

## 7. 告警规则

### 7.1 查询告警规则
- 方法：`GET /api/alert-rules`
- 返回：`AlertRule[]`

### 7.2 新增告警规则
- 方法：`POST /api/alert-rules`
- 请求体字段：
  - `deviceId`
  - `deviceCode` 服务端可回填
  - `deviceName` 服务端可回填
  - `dataType`
  - `operator`：`>` `>=` `<` `<=`
  - `threshold`
  - `enabled`
  - `messageTemplate`
  - `cooldownSeconds`

### 7.3 更新告警规则
- 方法：`PUT /api/alert-rules/{id}`

### 7.4 更新启停状态
- 方法：`PUT /api/alert-rules/{id}/status?enabled=1`

### 7.5 删除告警规则
- 方法：`DELETE /api/alert-rules/{id}`

### 7.6 告警行为说明
- 当前命中规则后仅执行控制台打印
- 打印内容至少应包含：规则 ID、设备编码、设备名称、数据项、阈值条件、当前值、触发时间、消息文本
- 冷却时间内不重复通知

## 8. OPC UA 接入
- `GET /api/opcua/status`：查询 OPC UA 服务状态
- `POST /api/opcua/device`：创建设备节点
- `POST /api/opcua/sync-devices`：同步设备到 OPC UA
- `DELETE /api/opcua/device/{deviceName}`：删除设备节点
- `PUT /api/opcua/device/{deviceName}/data`：更新节点数据
- `PUT /api/opcua/device/{deviceName}/status`：更新节点状态

## 9. 场景 JSON 兼容说明

### 9.1 新格式

```json
{
  "version": "1.0.0",
  "sceneMeta": {
    "id": 11,
    "name": "泵房场景",
    "description": "一期测试场景"
  },
  "skybox": {
    "enabled": true
  },
  "models": [
    {
      "id": "scene-model-1",
      "uuid": "scene-model-1",
      "name": "主泵模型",
      "file": "models/pump.glb",
      "position": { "x": 0, "y": 0, "z": 0 },
      "rotation": { "x": 0, "y": 0, "z": 0 },
      "scale": { "x": 1, "y": 1, "z": 1 },
      "userData": {
        "sceneModelId": "scene-model-1"
      }
    }
  ],
  "bindings": []
}
```

### 9.2 旧格式

```json
{
  "models": [],
  "skyboxEnabled": true
}
```

- 兼容策略：旧格式缺失字段时，由前端导入逻辑补默认值。
